    用 VS Code + 插件 + GitHub 写出你的图文笔记博客（零基础图文教程）

    零基础也能搞定！本教程将带你一步一步完成： - 本地写 Markdown 笔记 - 安装 VS Code 插件增强体验 - 配置 Git 并推送到 GitHub - 使用 GitHub Pages 发布成网页博客！

    第 1 部分：准备环境（VS Code + 插件 + GitHub 账号）

    1. 安装 VS Code

    • 什么是 VS Code？ Visual Studio Code（简称 VS Code） 是微软出品的一个免费、跨平台的轻量级代码编辑器,其特点如下：

      特性 说明
      多语言支持 支持 Markdown、Python、HTML 等上百种语言
      插件丰富 可安装扩展增强 Markdown、Git、代码提示等功能
      Git 集成 内建 Git 功能，适合版本控制

      因为它写 Markdown 非常方便，同时能与 Git 和 GitHub 完美集成，适合做“代码+文档+网页”型博客。 - 官网：https://code.visualstudio.com/ - 下载并安装，打开主界面

      2. 安装 Markdown 插件

      • 什么是 Markdown？ Markdown 是一种轻量级标记语言，主要用于写作和格式化文本。核心思想： 用纯文本语法（比如 # 表示标题，- 表示列表）来写结构化内容，写作体验简洁直观。优势：
      • 类可读、机器也易解析
      • 跨平台通用，写一遍可转为 HTML、PDF、Word
      • 被 GitHub、博客系统、写作平台广泛支持
      • 什么是插件系统： VS Code 的插件系统让它变得强大灵活。插件相当于给编辑器加“扩展功能”。插件就像手机 App，按需安装，用多少装多少。
      • 实现本文功能推荐安装的插件：
      • Markdown All in One：目录、快捷键等增强
      • Markdown Preview Enhanced：实时预览
      • markdownlint：语法检查
      • Paste Image: 在 Markdown 中粘贴截图（Ctrl + V）时，图片自动保存到指定文件夹（如 assets/images/），并在 .md 中自动生成对应路径的引用。（注意： 安装完毕之后需要设置图片存储位置：打开 VS Code 设置（快捷键：Ctrl + ,），搜索：pasteImage.path 修改为 assets/images/${currentFileName} 这会让所有粘贴的图片保存在当前 .md 文件所在目录下的 assets/images 子文件夹中, 并按日期分类存图。

      3. 安装Git

      • 什么是git? Git 是一种“版本控制系统”，可记录文件每次修改的历史，方便你回滚、更改、协作。其核心概念是：
      概念 含义
      工作区 你正在编辑的文件
      暂存区 git add 后的文件（准备提交）
      本地仓库 git commit 后保存的版本历史
      远程仓库 GitHub 等平台托管的副本

    • 为什么需要它？

    • 防止误删，方便版本回退
    • 可同步笔记到云端
    • 多人协作时可以分支管理冲突
    • 下载安装官网： https://git-scm.com/。安装后运行 git --version 检查是否成功

    4. 注册 GitHub 账号

    • 什么是GitHup？ GitHub 是全球最大的代码托管平台，基于 Git 技术构建。你可以把写好的 Markdown 文件推送到 GitHub：
    • 保留版本历史
    • 和别人共享/展示
    • 用 GitHub Pages 免费部署为网站
    • GitHup注册账号官网： https://github.com
